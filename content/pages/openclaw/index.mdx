# OpenClaw

This section is reserved for OpenClaw itself.

The goal is to keep OpenClaw-specific material in one place instead of scattering it across infrastructure notes just because some of it happens to run on Proxmox.

## Planned Focus

- architecture and system boundaries

- metrics, testing, and quality gates
- project evolution and operator-facing notes

If the main question is how to deploy or operate the surrounding infrastructure, that content belongs in [Proxmox](/pages/proxmox) or the relevant platform section. If the main question is how OpenClaw itself works, it belongs here.
