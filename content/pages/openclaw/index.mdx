# OpenClaw

OpenClaw has grown into its own little operating shape, so this page keeps the moving parts in one place instead of scattering them across infrastructure notes.

This section is the behavior layer: orchestrator, specialist agents, household workflows, voice and media handling, the task board, validation discipline, and the line between what is already live and what is still only a good idea.

The deployment notes still live elsewhere. That keeps the story honest and makes the project easier to read without repeating the same setup details in every page.

- [OpenClaw Architecture, Agents, And Workflows](/pages/openclaw/architecture-agents-and-workflows) - the current system view: commander, specialist agents, household routing, voice, board tasks, and how the pieces fit.
- [OpenClaw Workspace, Skills, And Use Cases](/pages/openclaw/workspace-skills-and-use-cases) - shape the per-agent workspace, choose the right extension surface, and decide which jobs belong in prompts, skills, or scripts.
- [OpenClaw Voice Transcription With whisper.cpp](/pages/openclaw/voice-transcription-with-whisper-cpp) - the working voice path for the current binary: `tools.media.audio`, a CLI bridge, and a local whisper.cpp server instead of dead-end handler experiments.
- [OpenClaw Workforce, Delegation, And Review Patterns](/pages/openclaw/workforce-delegation-and-review-patterns) - translate the older Mission Control workforce ideas into the current commander-era model, including delegation, reflexion, and research fan-out.
- [OpenClaw Operations And Troubleshooting](/pages/openclaw/operations-and-troubleshooting) - validate the gateway, router, board, heartbeats, memory, CalDAV, voice path, and channels in a fixed order.
