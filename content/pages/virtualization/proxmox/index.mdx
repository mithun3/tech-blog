# Proxmox Platform

These pages cover the Proxmox platform itself — how its components fit together and how to configure the pieces that every VM and container depends on.

## In This Section

- [Why I Use Proxmox](/pages/virtualization/proxmox/why-i-use-proxmox) — a practical, first-person explanation of why Proxmox works so well in a homelab and where it may still be overkill.
- [Proxmox Fundamentals](/pages/virtualization/proxmox/proxmox-fundamentals) — a reference-style overview of what Proxmox VE is, how Type 1 virtualization works, how VMs differ from LXCs, and where Proxmox fits against common alternatives.
- [Proxmox Stack](/pages/virtualization/proxmox/proxmox-stack) — how Proxmox layers QEMU, KVM, LXC, Linux storage, and networking into one workable homelab platform.
- [Storage and Snapshots](/pages/virtualization/proxmox/storage-and-snapshots) — VM disk formats, controller choices, snapshots, templates, and backup boundaries.
- [Networking Models](/pages/virtualization/proxmox/networking-models) — bridged, NAT, and isolated networking for VMs and containers, with guidance on when to use each.
- [Resource Allocation](/pages/virtualization/proxmox/resource-allocation) — practical sizing for CPU, memory, disk, and headroom across VMs and containers.

## How To Use It

Start with [Why I Use Proxmox](/pages/virtualization/proxmox/why-i-use-proxmox) if you want the subjective case first. Then read [Proxmox Fundamentals](/pages/virtualization/proxmox/proxmox-fundamentals) and [Proxmox Stack](/pages/virtualization/proxmox/proxmox-stack) for the structural view. After that, use the storage, networking, and resource pages as practical reference when building guests.
